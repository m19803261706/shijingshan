## 1. 项目概述

### 1.1 项目背景

为规范医疗安全不良事件的上报与管理流程，提升医院医疗质量安全管理水平，建设不良事件上报系统。系统采用非惩罚性、自愿上报原则，通过信息化手段实现事件上报、审核、处理、整改的全流程闭环管理。

### 1.2 建设目标

- 建立统一的不良事件上报平台，规范上报流程
- 实现事件分类流转，提高处理效率
- 形成"上报→审核→处理→整改→确认"的闭环管理机制
- 为医疗质量持续改进提供数据支撑

### 1.3 系统边界

| 范围 | 说明 |
|------|------|
| 在范围内 | 不良事件上报、审核、处理、整改管理、基础统计 |
| 不在范围内 | 投诉纠纷管理、满意度调查、HIS对接、移动端 |

## 2. 业务流程

### 2.1 核心流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  临床人员   │───▶│ 临床审核人员 │───▶│ 职能科室    │
│  上报事件   │    │  审核事件   │    │  处理事件   │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                             │
       ┌─────────────────────────────────────┤
       │                                     │
       ▼                                     ▼
┌─────────────┐                       ┌─────────────┐
│  临床人员   │◀──────────────────────│  要求整改   │
│  填写整改   │                       └─────────────┘
└──────┬──────┘
       │
       ▼
┌─────────────┐    ┌─────────────┐
│  提交整改   │───▶│ 科室确认    │───▶ 结案 / 再次整改
└─────────────┘    └─────────────┘
```

### 2.2 流程说明

| 节点 | 操作人 | 操作说明 |
|------|--------|----------|
| 上报 | 临床人员 | 选择事件分类，填写事件信息，提交上报 |
| 审核 | 临床审核人员 | 审核本科室上报的事件，通过或退回 |
| 处理 | 职能科室人员 | 查看分流至本科室的事件，要求整改或直接结案 |
| 整改 | 临床人员 | 根据整改要求填写整改措施，提交整改 |
| 确认 | 职能科室人员 | 确认整改完成结案，或要求再次整改 |

### 2.3 事件状态

| 状态 | 说明 |
|------|------|
| 草稿 | 已保存未提交 |
| 待审核 | 已提交，等待临床审核 |
| 已退回 | 审核退回，待修改重新提交 |
| 待处理 | 审核通过，等待职能科室处理 |
| 待整改 | 职能科室要求整改，等待临床填写 |
| 整改中 | 已提交整改，等待科室确认 |
| 已结案 | 处理完成，流程结束 |

## 3. 功能需求

### 3.1 事件上报

**使用角色**：临床人员

| 功能 | 说明 |
|------|------|
| 新建上报 | 创建新的不良事件上报单 |
| 选择分类 | 选择事件所属分类（医疗类/护理类等） |
| 填写信息 | 填写事件发生时间、地点、经过、涉及人员等 |
| 上传附件 | 上传相关图片或文件（可选） |
| 保存草稿 | 保存为草稿，可继续编辑 |
| 提交上报 | 提交至临床审核人员 |
| 我的上报 | 查看本人上报的事件列表及状态 |
| 修改重提 | 对退回的事件进行修改后重新提交 |

**上报表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 事件分类 | 下拉选择 | 是 | 医疗类/护理类/药品类等 |
| 发生科室 | 下拉选择 | 是 | 事件发生的科室 |
| 发生时间 | 日期时间 | 是 | 事件发生的时间 |
| 发生地点 | 文本 | 是 | 具体地点 |
| 事件级别 | 下拉选择 | 是 | Ⅰ级/Ⅱ级/Ⅲ级/Ⅳ级 |
| 事件经过 | 多行文本 | 是 | 详细描述事件经过 |
| 涉及人员 | 文本 | 否 | 涉及的医护人员 |
| 患者信息 | 文本 | 否 | 涉及的患者信息 |
| 初步原因 | 多行文本 | 否 | 事件发生的初步原因分析 |
| 附件 | 文件上传 | 否 | 支持图片、PDF等 |

### 3.2 科室审核

**使用角色**：临床审核人员

| 功能 | 说明 |
|------|------|
| 待审核列表 | 查看本科室待审核的事件 |
| 事件详情 | 查看上报事件的详细信息 |
| 审核通过 | 审核通过，流转至对应职能科室 |
| 审核退回 | 退回上报人，需填写退回原因 |
| 已审核列表 | 查看已审核的事件记录 |

### 3.3 职能科室处理

**使用角色**：职能科室人员

| 功能 | 说明 |
|------|------|
| 待处理列表 | 查看流转至本科室的事件 |
| 事件详情 | 查看事件详细信息及流转记录 |
| 要求整改 | 下发整改要求，填写整改意见 |
| 直接结案 | 无需整改，直接结案处理 |
| 待确认列表 | 查看已提交整改待确认的事件 |
| 确认整改 | 确认整改完成，事件结案 |
| 再次整改 | 整改不到位，要求再次整改 |
| 已结案列表 | 查看已结案的事件记录 |

**整改要求字段**：

| 字段 | 类型 | 必填 |
|------|------|------|
| 整改意见 | 多行文本 | 是 |
| 整改期限 | 日期 | 是 |
| 备注 | 多行文本 | 否 |

### 3.4 整改管理

**使用角色**：临床人员

| 功能 | 说明 |
|------|------|
| 待整改列表 | 查看需要整改的事件 |
| 整改要求 | 查看职能科室下发的整改意见 |
| 填写整改 | 填写整改措施及完成情况 |
| 提交整改 | 提交整改至职能科室确认 |
| 整改记录 | 查看历次整改记录 |

**整改反馈字段**：

| 字段 | 类型 | 必填 |
|------|------|------|
| 整改措施 | 多行文本 | 是 |
| 完成情况 | 多行文本 | 是 |
| 附件 | 文件上传 | 否 |

### 3.5 事件查询

**使用角色**：所有用户（按权限过滤数据）

| 功能 | 说明 |
|------|------|
| 条件查询 | 按时间、科室、分类、状态等条件筛选 |
| 事件详情 | 查看事件完整信息及流转记录 |
| 导出Excel | 导出查询结果 |

### 3.6 统计分析

**使用角色**：管理人员、职能科室人员

| 功能 | 说明 |
|------|------|
| 按科室统计 | 各科室上报数量统计 |
| 按分类统计 | 各分类事件数量统计 |
| 按时间统计 | 按月/季度/年统计趋势 |
| 按状态统计 | 各状态事件数量统计 |
| 报表导出 | 导出统计报表 |

### 3.7 系统管理

**使用角色**：系统管理员

| 功能 | 说明 |
|------|------|
| 用户管理 | 用户账号的增删改查、密码重置 |
| 角色管理 | 角色定义及权限配置 |
| 科室管理 | 科室信息维护 |
| 分类管理 | 事件分类维护，配置分类与职能科室对应关系 |
| 操作日志 | 查看系统操作日志 |

## 4. 角色权限

### 4.1 角色定义

## 5. 事件分类

### 5.1 分类与科室对应

| 分类名称 | 对应职能科室 | 说明 |
|----------|-------------|------|
| 医疗类 | 医务科 | 诊疗相关不良事件 |
| 护理类 | 护理部 | 护理相关不良事件 |
| 药品类 | 药剂科 | 药品相关不良事件 |
| 器械类 | 设备科 | 医疗器械相关不良事件 |
| 感染类 | 院感科 | 医院感染相关不良事件 |
| 输血类 | 输血科 | 输血相关不良事件 |
| 后勤类 | 后勤保障部 | 后勤保障相关不良事件 |
| 其他类 | 医务科 | 其他类型不良事件 |

> 注：分类可根据医院实际情况进行配置调整。

### 5.2 事件级别

| 级别 | 名称 | 说明 |
|------|------|------|
| Ⅰ级 | 警告事件 | 非预期的死亡或重大永久性功能丧失 |
| Ⅱ级 | 不良后果事件 | 对患者造成中等程度伤害 |
| Ⅲ级 | 未造成后果事件 | 已发生但未对患者造成伤害 |
| Ⅳ级 | 隐患事件 | 未发生但存在隐患 |
